Linux 进程外部函数与动态库依赖分析工具。

`process-deps-analyzer` 用于分析 **正在运行的 Linux 进程**，识别进程调用的外部函数以及动态符号依赖，并自动解析这些符号来自哪个共享库。

工具通过解析 ELF 信息（重定位表和动态符号表）来生成结构化分析结果，适用于依赖分析、系统调试、安全审计和嵌入式系统分析等场景。

---

## 功能

- 分析进程调用的 **外部函数 (JMP_SLOT / PLT)**
- 解析 **动态符号表中的外部符号引用**
- 自动识别 **符号所属的共享库**
- 支持 **多个同名进程实例**
- 生成 **CSV 分析结果**
- 自动生成 **分析汇总报告**
- 使用 **符号缓存机制提升分析速度**

---

## 工作原理

脚本通过分析 ELF 二进制信息获取外部依赖：

1. **外部函数调用分析**

   通过解析 ELF **重定位表 (JMP_SLOT)** 识别程序调用的外部函数。

2. **动态符号依赖分析**

   通过解析 **动态符号表 (Dynamic Symbol Table)** 获取未定义符号（UND），识别程序引用的外部符号。

3. **符号归属库解析**

   - 使用 `ldd` 获取进程依赖库
   - 使用 `nm -D` 提取库导出符号
   - 构建符号缓存以快速匹配符号所属库
